
# AIアプリ開発ナレッジ共有Webアプリ 構想まとめ

## 1. 目的

DX推進担当者と情報支援グループが、AIアプリ開発に関する情報を共有・蓄積するためのWebアプリを構築する。

単なる掲示板やチャットではなく、以下を一元管理できる基盤とする。

* 参考になる外部URLの共有
* X投稿、YouTube、技術記事、GitHub、公式ドキュメント等の共有
* AIアプリ開発に関する質問・回答
* 検証メモ・試行結果の蓄積
* 担当者が作成したアプリURLの共有
* 作成アプリに対するコメント・レビュー
* タグやプロジェクト単位での整理・検索

---

## 2. コンセプト

一言で表すと、

> **AIアプリ開発版の「チーム内ナレッジブックマーク＋Q&A＋検証メモ＋アプリレビュー」**

です。

SlackやTeamsのようなチャット・SNS型ではなく、
**情報が流れず、後から検索・再利用できるナレッジ共有基盤**として設計する。

---

## 3. 主な利用者

| 利用者       | 主な利用内容                     |
| --------- | -------------------------- |
| DX推進担当者   | AI活用・AIアプリ開発に関する情報収集、共有、質問 |
| 情報支援グループ  | 技術検証、開発ノウハウ共有、セキュリティ確認     |
| 管理者       | ユーザー管理、タグ管理、投稿管理           |
| 必要に応じた関係者 | 作成アプリの確認、コメント、改善提案         |

## 4. 主な機能

## ① 外部URL共有機能

X、YouTube、技術記事、GitHub、公式ドキュメントなどのURLを登録・共有する機能。

登録項目例：

| 項目     | 内容                           |
| ------ | ---------------------------- |
| URL    | 共有したいリンク                     |
| タイトル   | 自動取得または手入力                   |
| 種別     | X、YouTube、記事、GitHub、公式Docsなど |
| サムネイル  | 可能であれば自動取得                   |
| 概要     | 自動取得または手入力                   |
| 共有コメント | なぜ参考になるのか                    |
| タグ     | RAG、Dify、API、セキュリティなど        |
| 重要度    | 高・中・低                        |
| ステータス  | 未確認、確認中、検証済み、採用候補、保留         |

重要なのは、URLだけでなく、
**「なぜ共有したのか」「どの開発に役立つのか」** を残すこと。

---

## ② Q&A機能

AIアプリ開発に関する疑問を投稿し、回答を蓄積する機能。

質問例：

* RAGでPDFを扱うときのチャンクサイズは？
* DifyとLangChainはどう使い分けるか？
* 学内認証とAIアプリをどう連携するか？
* YouTube動画を要約してナレッジ化できるか？
* 外部API利用時の注意点は？

質問・回答はタグ付けして、後から検索できるようにする。

---

## ③ 検証メモ機能

調べた情報やツールを実際に試した結果を記録する機能。

記録項目例：

| 項目       | 内容                 |
| -------- | ------------------ |
| タイトル     | 検証テーマ              |
| 目的       | 何を確認したかったか         |
| 試したこと    | 設定、手順、条件           |
| 結果       | うまくいった点、課題         |
| 結論       | 採用可否、今後の対応         |
| 関連URL    | 参考記事、動画、GitHub等    |
| 関連プロジェクト | FAQ検索アプリ、議事録要約アプリ等 |

この機能により、単なる情報共有ではなく、
**実際に試したノウハウ**として蓄積できる。

---

## ④ 作成アプリURL共有機能

担当者が作成したAIアプリのURLを共有する機能。

登録項目例：

| 項目     | 内容                               |
| ------ | -------------------------------- |
| アプリ名   | 議事録要約アプリ、FAQ検索アプリなど              |
| アプリURL | 実際にアクセスするURL                     |
| 概要     | 何をするアプリか                         |
| 作成者    | 担当者名                             |
| 開発目的   | どの課題を解決するために作ったか                 |
| 使用技術   | Dify、Next.js、Python、OpenAI APIなど |
| 利用AI   | ChatGPT API、Claude、Geminiなど      |
| 利用範囲   | 個人検証、グループ内、関係者限定など               |
| ステータス  | 試作、検証中、利用中、改善中、停止中               |
| 注意事項   | 入力禁止情報、既知の不具合など                  |
| 関連資料   | GitHub、仕様メモ、参考URLなど              |

この機能は必須とする。

---

## ⑤ 作成アプリへのコメント・レビュー機能

共有されたアプリに対して、関係者がコメントを残せる機能。

コメント種別例：

| 種別       | 内容                  |
| -------- | ------------------- |
| 感想       | 使いやすい、便利、わかりやすい     |
| 改善提案     | 項目を追加したい、画面を変えたい    |
| 不具合報告    | エラーが出る、表示が崩れる       |
| 活用アイデア   | 他部署でも使えそう、研修で紹介できそう |
| 技術メモ     | 実装方法、API制限、モデル変更の影響 |
| セキュリティ指摘 | 入力情報、ログ、権限管理の懸念     |

アプリ開発後の改善サイクルを回すために重要な機能。

---

## ⑥ タグ・カテゴリ管理

情報を探しやすくするため、タグで分類する。

タグ例：

| 分類     | タグ例                                                |
| ------ | -------------------------------------------------- |
| 技術     | RAG、ベクトルDB、API、OCR、Function Calling、Agents         |
| ツール    | Dify、LangChain、LlamaIndex、OpenAI API、Claude、Gemini |
| 開発     | Next.js、Python、FastAPI、PostgreSQL、Docker、GitHub    |
| 用途     | FAQ検索、議事録要約、問い合わせ対応、文書検索、研修資料作成                    |
| セキュリティ | 個人情報、認証、権限管理、ログ、プロンプトインジェクション                      |
| 状態     | 未確認、検証中、検証済み、採用候補、見送り                              |

---

## ⑦ プロジェクト別整理機能

URL、質問、検証メモ、作成アプリをプロジェクト単位で紐づける。

プロジェクト例：

* 学内FAQ検索アプリ
* 議事録要約アプリ
* 問い合わせ分類AI
* 業務マニュアル検索AI
* 研修資料作成支援アプリ
* 文書要約アプリ

これにより、

> このプロジェクトに関係する参考URL、質問、検証結果、試作アプリをまとめて確認する

ことができる。

## 10. 運用イメージ

情報共有の流れは以下。

```text
参考になるURLを見つける
↓
URLとコメントを登録する
↓
関係者が確認・コメントする
↓
必要に応じて検証メモを作成する
↓
試作アプリを作成する
↓
アプリURLを共有する
↓
関係者が使ってコメント・改善提案を残す
↓
ノウハウとして蓄積される
```

---

## 11. 重要な設計方針

このWebアプリでは、次の考え方を重視する。

### チャット型にしない

情報が流れてしまうため、SNSやチャットのような設計にはしない。
投稿ごとにタグ・ステータス・関連プロジェクトを持たせ、後から検索しやすくする。

### URLだけで終わらせない

URLに加えて、必ず以下を残す。

```text
なぜ共有したのか
何に使えそうか
誰が確認したか
試した結果どうだったか
採用する価値があるか
```

### 作成アプリを中心に改善サイクルを回す

特に重要なのは、作ったAIアプリを共有して、コメントや改善提案を集めること。

```text
作る
↓
共有する
↓
使ってもらう
↓
コメントを残す
↓
改善する
↓
ノウハウ化する
```

## 12. 最終コンセプト

最終的には、以下のようなWebアプリとして整理できます。

> **DX推進担当者と情報支援グループが、AIアプリ開発に関する外部情報、質問、検証結果、作成アプリ、レビューコメントを蓄積し、検索・再利用できるナレッジ共有Webアプリ。**

特に重視する機能は、

> **URL共有、Q&A、検証メモ、作成アプリ共有、コメント・レビュー**

です。

これにより、単なるリンク集ではなく、
**AIアプリ開発の実践知を組織内に蓄積する基盤**になります。
