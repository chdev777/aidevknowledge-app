今回のWebアプリでは、単なる共有場所ではなく、各担当者が自分の情報を整理し、必要に応じて共有できる仕組みにすると使いやすくなります。

 

# 追加機能：マイページ

 

## 1. 位置づけ

 

マイページは、ログインしている利用者ごとの専用ページです。

 

利用者は、まず自分用に情報を登録できます。
その後、必要に応じて「共有」に切り替えることで、DX推進担当者・情報支援グループのメンバーに公開されます。

 

つまり、

 

> **自分用のメモ・URL・アプリ情報を蓄積し、共有したいものだけチームに公開できる機能**

 

です。

 

---

 

## 2. 公開状態の考え方

 

登録した情報には、公開状態を持たせます。

 

| 公開状態 | 見える範囲                   |
| ---- | ----------------------- |
| 非公開  | 登録した本人のみ                |
| 共有   | DX推進担当者・情報支援グループ内で閲覧可能  |
| 限定共有 | 指定したメンバーまたはプロジェクト関係者のみ  |
| 公開停止 | 以前共有していたが、現在は非公開または閲覧停止 |

 

最初はシンプルに、

 

```text
非公開 / 共有
```

 

の2段階で始めてもよいです。

 

---

 

## 3. マイページでできること

 

マイページでは、ログイン利用者が自分の登録情報を一覧管理できます。

 

対象は以下です。

 

* 自分が登録した外部URL
* 自分が投稿した質問
* 自分が書いた回答
* 自分が作成した検証メモ
* 自分が登録した作成アプリ
* 自分が書いたコメント
* お気に入り登録した情報
* 下書き中の投稿
* 非公開メモ

## 6. 登録時の公開設定

 

URLやメモ、作成アプリを登録するときに、公開状態を選べるようにします。

 

```text
URLを登録する

 

URL：
[ https://example.com ]

 

共有コメント：
[ あとで検証したい記事。RAG設計に使えそう。 ]

 

公開設定：
( ) 非公開：自分だけが見られる
( ) 共有：DX推進担当者・情報支援グループに共有する
```

 

初期値は **非公開** にするのが安全です。

 

ただし、チームで積極的に共有したい運用なら、初期値を **共有** にする選択肢もあります。
安全重視なら「非公開」がよいです。

 

---

 

# 7. 共有・非公開の切り替え

 

登録後に、いつでも公開状態を変更できるようにします。

 

```text
操作：
[共有する]
[非公開に戻す]
```

 

共有に切り替えた場合：

 

```text
この情報を共有しますか？

 

共有すると、DX推進担当者・情報支援グループのメンバーが閲覧できます。

 

[共有する] [キャンセル]
```

 

非公開に戻す場合：

 

```text
この情報を非公開にしますか？

 

非公開にすると、他の利用者は閲覧できなくなります。
ただし、過去に付いたコメントの扱いは管理方針に従います。

 

[非公開にする] [キャンセル]
```

 

---

 

## 8. コメントとの関係

 

共有された情報には、他のメンバーがコメントできます。

 

ただし、あとで非公開に戻した場合の扱いを決めておく必要があります。

 

おすすめは以下です。

 

| 状態      | コメントの扱い            |
| ------- | ------------------ |
| 共有中     | 他メンバーがコメント可能       |
| 非公開に戻した | 他メンバーは本文・コメントを閲覧不可 |
| 再共有した   | 以前のコメントも再表示        |
| 削除した    | 本文・コメントとも削除または論理削除 |

 

シンプルにするなら、

 

> 非公開に戻すと、他者コメントも含めて本人以外には見えなくなる

 

でよいです。

 

---

 

# 9. 権限設計

 

最低限、以下のルールにします。

 

| 操作         |   本人 | 他メンバー |     管理者 |
| ---------- | ---: | ----: | ------: |
| 非公開投稿を見る   |    可 |    不可 |  可または不可 |
| 共有投稿を見る    |    可 |     可 |       可 |
| 非公開投稿を編集   |    可 |    不可 | 必要に応じて可 |
| 共有投稿を編集    |    可 |    不可 | 必要に応じて可 |
| 共有投稿にコメント  |    可 |     可 |       可 |
| 非公開投稿にコメント |    可 |    不可 |  可または不可 |
| 公開状態を変更    | 本人のみ |    不可 | 必要に応じて可 |
| 削除         | 本人のみ |    不可 | 必要に応じて可 |

 

管理者が非公開情報を閲覧できるかどうかは、運用方針として決める必要があります。
個人メモの性格を強くするなら、管理者も通常画面では見えない設計が望ましいです。

# 11. 一覧画面との関係

 

通常の「ナレッジ一覧」には、共有されたものだけ表示します。

 

```text
ナレッジ一覧
→ visibility = shared のものだけ表示
```

 

マイページには、自分が登録したものをすべて表示します。

 

```text
マイページ
→ created_by = 自分 のものを表示
→ private / shared の両方を表示
```

 

検索も同じ考え方にします。

 

## 通常検索

 

```text
共有されている情報だけ検索対象
```

 

## マイページ検索

 

```text
自分の非公開情報も検索対象
```

# まとめ

 

マイページ機能を追加すると、このWebアプリは次のような使い方になります。

 

> 各担当者がまず自分専用の場所に、URL、メモ、質問、作成アプリを登録する。
> 共有したいものだけ「共有」に切り替えることで、DX推進担当者・情報支援グループ内に公開される。
> 共有された情報には、他メンバーがコメントやレビューを残せる。
> 非公開にすれば、自分だけが見られる状態に戻せる。

 

この設計にすると、利用者は気軽に情報を蓄積でき、チームには必要な情報だけを共有できます。
